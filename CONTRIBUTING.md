
# Contributing to the Vedic AI Sovereign Kernel

We welcome contributions to the Vedic AI Sovereign Kernel project! By participating, you agree to abide by our Code of Conduct.

## How to Contribute

There are many ways to contribute, from reporting bugs to suggesting new features or directly implementing code.

### Reporting Bugs

If you find a bug, please open an issue on the [GitHub Issues page](https://github.com/divineearthly/Vedic-AI-Sovereign-Kernel/issues) and provide the following information:

*   A clear and concise description of the bug.
*   Steps to reproduce the behavior.
*   Expected behavior.
*   Actual behavior.
*   Screenshots or logs if applicable.
*   Your environment details (OS, Python version, `llama.cpp` version, etc.).

### Suggesting Enhancements

We love new ideas! If you have a suggestion for an enhancement or a new feature:

1.  Check the [GitHub Issues page](https://github.com/divineearthly/Vedic-AI-Sovereign-Kernel/issues) to see if it has already been suggested.
2.  If not, open a new issue and clearly describe your idea, including why you think it would be beneficial to the project.

### Code Contributions

For code contributions, please follow these steps:

1.  **Fork the Repository**: Start by forking the `Vedic-AI-Sovereign-Kernel` repository to your GitHub account.
2.  **Clone Your Fork**: Clone your forked repository to your local machine:
    ```bash
    git clone https://github.com/<YOUR_GITHUB_USERNAME>/Vedic-AI-Sovereign-Kernel.git
    cd Vedic-AI-Sovereign-Kernel
    ```
3.  **Create a New Branch**: Create a new branch for your feature or bug fix. Use a descriptive name (e.g., `feature/add-sutra-X` or `bugfix/issue-Y`):
    ```bash
    git checkout -b your-feature-branch-name
    ```
4.  **Make Your Changes**: Implement your changes, adhering to the project's coding style and principles (e.g., Vedic Sutra integration guidelines).
5.  **Test Your Changes**: Ensure your changes do not introduce new bugs and ideally include new tests for your feature or fix.
6.  **Commit Your Changes**: Write clear and concise commit messages. Reference the issue number if applicable (e.g., `git commit -m "Feat: Implement Sutra X for Y [fixes #123]"`).
7.  **Push to Your Fork**: Push your new branch to your forked repository on GitHub:
    ```bash
    git push origin your-feature-branch-name
    ```
8.  **Open a Pull Request**: Go to the original `Vedic-AI-Sovereign-Kernel` repository on GitHub and open a Pull Request (PR) from your branch to the `main` branch. Provide a detailed description of your changes and why they are needed.

## Coding Style

*   Follow C++20 standards for C++ code.
*   Adhere to PEP 8 for Python code.
*   Ensure code is well-commented and documented.
*   Prioritize resource efficiency and alignment with Vedic principles, especially for mobile platforms.

## Code of Conduct

This project adheres to the Contributor Covenant Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to divineearthly@example.com.
